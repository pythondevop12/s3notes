## ⚡ Quick Theory — Before We Start

### What is an Object?
- Any file you store in S3 — image, PDF, video, JSON, zip, anything
- Every object has 3 parts:
```
1. Data      →  the actual file content
2. Key       →  the unique name/path  (e.g. photos/user1.jpg)
3. Metadata  →  information about the file (size, type, custom tags)
```

### Object Key — Important to Understand
```
photos/profile/user1.jpg   →  looks like a folder path
                              but S3 has NO real folders
                              "photos/profile/" is just part of the key name

Bucket: my-app
Key:    photos/profile/user1.jpg
URL:    https://my-app.s3.ap-south-1.amazonaws.com/photos/profile/user1.jpg
```

### Storage Class — Set at Upload Time
```
S3 Standard          →  default, frequently accessed files
S3 Standard-IA       →  infrequent access, cheaper
S3 Glacier           →  archival, very cheap, slow retrieval
S3 Intelligent-Tiering →  auto-moves based on access pattern
```
> We deep dive on storage classes in Video 04. For now — leave it as **Standard**.

## 🖥️ Console Demo — Upload a File

### Step 1 — Open your bucket
```
AWS Console → S3 → click your bucket name
```

### Step 2 — Upload a file (drag & drop)
1. Click **Upload**
2. Click **Add files** or drag files directly into the window
3. You'll see the file listed with size and type

### Step 3 — Set properties (optional but important to know)

**Storage class** — leave as Standard for now

**Metadata** — extra information attached to the object
```
System metadata (auto-set by AWS):
  Content-Type    →  image/jpeg, application/pdf, etc.
  Content-Length  →  file size in bytes
  Last-Modified   →  when it was uploaded

User-defined metadata (you set these):
  Key: x-amz-meta-author   Value: john
  Key: x-amz-meta-project  Value: s3-course
```

**Tags** — key-value labels for organising and billing
```
Key: Environment   Value: dev
Key: Team          Value: backend
```
> 💡 Tags are powerful for cost tracking. You can filter AWS bills by tag.

### Step 4 — Click Upload

You'll see a green success banner. File is now in S3.

## 🔍 Explore the Uploaded Object

Click the file name in the bucket. You'll see:

```
Overview tab:
  Object URL     →  direct link (will return Access Denied — file is private)
  Key            →  full path/name of the object
  Size           →  file size
  Last modified  →  upload timestamp
  ETag           →  MD5 hash of the file (used for integrity checks)
  Storage class  →  Standard / IA / Glacier etc.

Properties tab:
  Encryption     →  SSE-S3 (default, encrypted at rest)
  Metadata       →  all metadata attached to this object
  Tags           →  all tags attached to this object
```

### Object URL — Why it says Access Denied
```
Your bucket has Block Public Access = ON
So the URL returns 403 Access Denied — this is CORRECT and SECURE

To share a file privately → use a Pre-Signed URL (covered in Video 09)
To make a file public    → change bucket policy (covered in Video 06)
```

## 📤 Upload via AWS CLI

### Single file upload
```bash
aws s3 cp myfile.txt s3://your-bucket-name/
```

### Upload to a specific folder (prefix)
```bash
aws s3 cp myfile.txt s3://your-bucket-name/documents/
```

### Upload with storage class
```bash
aws s3 cp myfile.txt s3://your-bucket-name/ --storage-class STANDARD_IA
```

### Upload with metadata
```bash
aws s3 cp myfile.txt s3://your-bucket-name/ \
  --metadata author=john,project=s3-course
```

### Upload with tags
```bash
aws s3 cp myfile.txt s3://your-bucket-name/ \
  --tagging "Environment=dev&Team=backend"
```

### Upload entire folder (sync)
```bash
# Upload whole local folder to S3
aws s3 sync ./my-local-folder s3://your-bucket-name/folder/

# Sync and delete files in S3 that no longer exist locally
aws s3 sync ./my-local-folder s3://your-bucket-name/folder/ --delete
```

> 💡 `sync` is smarter than `cp` — it only uploads files that changed or are new.
> Perfect for deploying static websites or backing up a folder.

## 📥 Downloading Files

### Via Console
1. Click the object name
2. Click **Download** button (top right)
3. Or click **Open** to preview in browser (if file type supports it)

### Via CLI — single file
```bash
aws s3 cp s3://your-bucket-name/myfile.txt ./downloads/
```

### Via CLI — download entire folder
```bash
aws s3 sync s3://your-bucket-name/folder/ ./local-folder/
```

### Via CLI — download entire bucket
```bash
aws s3 sync s3://your-bucket-name/ ./local-backup/
```

## 📋 Managing Objects

### List objects in a bucket
```bash
# List all objects
aws s3 ls s3://your-bucket-name/

# List objects in a specific folder/prefix
aws s3 ls s3://your-bucket-name/photos/

# List recursively (all nested objects)
aws s3 ls s3://your-bucket-name/ --recursive

# List with file sizes in human-readable format
aws s3 ls s3://your-bucket-name/ --recursive --human-readable --summarize
```

### Copy an object (within S3, no download needed)
```bash
# Copy within same bucket
aws s3 cp s3://your-bucket/file.txt s3://your-bucket/backup/file.txt

# Copy to different bucket
aws s3 cp s3://bucket-1/file.txt s3://bucket-2/file.txt
```

### Move an object
```bash
aws s3 mv s3://your-bucket/old-location/file.txt s3://your-bucket/new-location/file.txt
```

### Delete an object
```bash
# Delete single object
aws s3 rm s3://your-bucket-name/myfile.txt

# Delete all objects with a prefix
aws s3 rm s3://your-bucket-name/old-files/ --recursive
```

### Update metadata (requires re-upload)
```bash
# You can't edit metadata in-place — copy the object onto itself with new metadata
aws s3 cp s3://your-bucket/file.txt s3://your-bucket/file.txt \
  --metadata-directive REPLACE \
  --metadata author=jane,version=2
```

## 🏷️ Tagging Objects

### Add tags via console
```
Click object → Tags tab → Edit → Add tag → Save
```

### Add tags via CLI
```bash
aws s3api put-object-tagging \
  --bucket your-bucket-name \
  --key myfile.txt \
  --tagging '{"TagSet":[{"Key":"Environment","Value":"prod"},{"Key":"Team","Value":"backend"}]}'
```

### View tags via CLI
```bash
aws s3api get-object-tagging \
  --bucket your-bucket-name \
  --key myfile.txt
```

> 💡 Max 10 tags per object. Tags are free to add.

## 🔄 cp vs sync — When to Use Which

| Command | Use when |
|---------|----------|
| `aws s3 cp` | Uploading/downloading a single file |
| `aws s3 sync` | Uploading/downloading a folder or keeping two locations in sync |
| `aws s3 mv` | Moving an object to a different location |
| `aws s3 rm` | Deleting objects |

## 🛠️ Useful Flags

```bash
--recursive          →  apply command to all objects in a prefix
--exclude "*.log"    →  skip files matching this pattern
--include "*.jpg"    →  only include files matching this pattern
--dryrun             →  preview what would happen without actually doing it
--quiet              →  suppress output (good for scripts)
--sse               →  enable server-side encryption on upload

# Example: sync only jpg files, skip everything else
aws s3 sync ./photos s3://my-bucket/photos/ \
  --exclude "*" \
  --include "*.jpg"
```

## ⭐ Key Takeaways

```
Object    =  file + key + metadata
Key       =  unique name, looks like a path, no real folders
Upload    =  console drag & drop  OR  aws s3 cp
Sync      =  aws s3 sync  →  only transfers what changed (smarter than cp)
Download  =  console Download button  OR  aws s3 cp / sync
Metadata  =  system (auto) + user-defined (x-amz-meta-*)
Tags      =  key-value labels, max 10 per object, useful for billing
Private   =  all objects private by default, URL returns 403
```
