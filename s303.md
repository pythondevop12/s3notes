# 📦 AWS S3 —: Uploading, Downloading & Managing Objects


---

### What is an Object?
- Any file you store in S3 — image, PDF, video, JSON, zip, anything
- Every object has 3 parts:
```
1. Data      →  the actual file content
2. Key       →  the unique name/path  (e.g. photos/user1.jpg)
3. Metadata  →  information about the file (size, type, custom tags)
```

### Object Key — Important to Understand
```
photos/profile/user1.jpg   →  looks like a folder path
                              but S3 has NO real folders
                              "photos/profile/" is just part of the key name

Bucket: my-app
Key:    photos/profile/user1.jpg
URL:    https://my-app.s3.ap-south-1.amazonaws.com/photos/profile/user1.jpg
```

### Storage Class — Set at Upload Time
```
S3 Standard          →  default, frequently accessed files
S3 Standard-IA       →  infrequent access, cheaper
S3 Glacier           →  archival, very cheap, slow retrieval
S3 Intelligent-Tiering →  auto-moves based on access pattern
```

# 📦 AWS S3 CLI Notes (Quick Reference)

---

## 📤 Upload Commands

### Upload single file
```bash
aws s3 cp myfile.txt s3://your-bucket-name/
````

### Upload to folder (prefix)

```bash
aws s3 cp myfile.txt s3://your-bucket-name/documents/
```

### Upload with storage class

```bash
aws s3 cp myfile.txt s3://your-bucket-name/ \
  --storage-class STANDARD_IA
```

### Upload with metadata

```bash
aws s3 cp myfile.txt s3://your-bucket-name/ \
  --metadata author=john,project=s3-course
```

### Upload with tags

```bash
aws s3 cp myfile.txt s3://your-bucket-name/ \
  --tagging "Environment=dev&Team=backend"
```

---

## 📂 Folder Upload / Sync

### Sync local folder → S3

```bash
aws s3 sync ./my-folder s3://your-bucket-name/folder/
```

### Sync with delete (mirror exact state)

```bash
aws s3 sync ./my-folder s3://your-bucket-name/folder/ --delete
```

---

## 📥 Download Commands

### Download single file

```bash
aws s3 cp s3://your-bucket-name/myfile.txt ./downloads/
```

### Download folder

```bash
aws s3 sync s3://your-bucket-name/folder/ ./local-folder/
```

### Download entire bucket

```bash
aws s3 sync s3://your-bucket-name/ ./local-backup/
```

---

## 📋 List Objects

```bash
aws s3 ls s3://your-bucket-name/
aws s3 ls s3://your-bucket-name/ --recursive
aws s3 ls s3://your-bucket-name/ --recursive --human-readable --summarize
```

---

## 📄 Copy / Move / Delete

### Copy (within S3)

```bash
aws s3 cp s3://bucket/file.txt s3://bucket/backup/file.txt
```

### Move object

```bash
aws s3 mv s3://bucket/file.txt s3://bucket/new/file.txt
```

### Delete single file

```bash
aws s3 rm s3://bucket/file.txt
```

### Delete folder

```bash
aws s3 rm s3://bucket/folder/ --recursive
```

---

## 🏷️ Tagging

### Add tags

```bash
aws s3api put-object-tagging \
  --bucket your-bucket-name \
  --key file.txt \
  --tagging '{"TagSet":[{"Key":"Environment","Value":"prod"}]}'
```

### Get tags

```bash
aws s3api get-object-tagging \
  --bucket your-bucket-name \
  --key file.txt
```

# 🔥 cp vs sync

## 🟢 aws s3 cp

* Best for **single file**
* Always copies (no change detection)
* Simple and direct

```bash
aws s3 cp file.txt s3://bucket/
```

👉 Use when: uploading/downloading one file

---

## 🔵 aws s3 sync

* Best for **folders**
* Only transfers **new or changed files**
* Faster for large data
* Can mirror using `--delete`

```bash
aws s3 sync ./folder s3://bucket/folder/
```

👉 Use when: backup, deployment, syncing directories

---

## ⚔️ Difference Table

| Feature            | cp              | sync              |
| ------------------ | --------------- | ----------------- |
| Use case           | Single file     | Folder            |
| Checks changes     | ❌ No            | ✅ Yes             |
| Speed              | ❌ Slower (bulk) | ✅ Faster          |
| Delete extra files | ❌ No            | ✅ With `--delete` |
