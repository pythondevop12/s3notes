# ☁️ AWS S3 — Complete Notes

> **Amazon Simple Storage Service** · Object Storage · Infinitely Scalable · 99.999999999% Durable

---

## 📌 What is S3?

- **S3 = Simple Storage Service** — object storage built for the internet
- Store and retrieve **any amount of data**, at **any time**, from **anywhere**
- S3 is a **global service** — but buckets are **regional**
- Used by Netflix, Airbnb, NASA, Dropbox, and millions of apps worldwide

### Object Storage vs Others

| Type | Example | Access | Best For |
|------|---------|--------|----------|
| **Object Storage** | S3 | HTTP URL | Files, backups, media, data lakes |
| **File Storage** | EFS, NAS | File path | Shared folders, home dirs |
| **Block Storage** | EBS, SSD | Raw blocks | Databases, OS volumes |

## 🪣 Core Concepts

### Bucket
- Top-level container for objects
- **Globally unique name** — no two buckets on AWS can share a name
- Lives in a specific **AWS region**
- Name rules: 3–63 chars, lowercase only, no underscores, no IP format

### Object
- Any file stored in S3 — photo, video, PDF, JSON, zip, etc.
- Max size: **5 TB** per object
- Upload > 5 GB → must use **multipart upload**
- Minimum size: **0 bytes**

### Key
- The **unique identifier** of an object inside a bucket
- Looks like a path: `profile/user1.jpg` — but S3 has **no real folders**
- The slash `/` is just part of the key name — it's all flat storage

### S3 URL Structure
```
https://{bucket-name}.s3.{region}.amazonaws.com/{key}

Example:
https://my-app-photos.s3.ap-south-1.amazonaws.com/profile/user1.jpg
         └─ bucket ──┘    └─ region ─┘              └──── key ─────┘
```

## 🗄️ Storage Classes

| Class | Use Case | Retrieval | Min Duration | Cost |
|-------|----------|-----------|--------------|------|
| **S3 Standard** | Frequently accessed data | Instant | None | $$$ |
| **S3 Standard-IA** | Infrequent access, rapid retrieval | Instant | 30 days | $$ |
| **S3 One Zone-IA** | Non-critical, infrequent | Instant | 30 days | $ |
| **S3 Intelligent-Tiering** | Unknown access patterns | Instant | None | Auto |
| **S3 Glacier Instant** | Archive, accessed once/quarter | Instant | 90 days | $ |
| **S3 Glacier Flexible** | Archive, 1–5 min to 12 hr retrieval | Minutes–Hours | 90 days | $$ |
| **S3 Glacier Deep Archive** | Long-term archive, 12–48 hr | Hours | 180 days | Cheapest |

### Decision Tree — Which Class?
```
Accessed frequently?
├── YES → S3 Standard
└── NO → Accessed occasionally?
          ├── YES → Need multiple AZs?
          │         ├── YES → Standard-IA
          │         └── NO  → One Zone-IA
          └── NO → Archive?
                    ├── Need instant retrieval? → Glacier Instant
                    ├── Can wait minutes?       → Glacier Flexible
                    └── Can wait hours?         → Glacier Deep Archive

Unknown pattern? → Intelligent-Tiering (auto-moves for you)
```

## 🔒 Security

### 3 Ways to Control Access

```
1. Bucket Policy      → JSON policy on the bucket (resource-based)
2. IAM Policy         → Attached to users/roles (identity-based)
3. ACLs               → Legacy, mostly disabled now
```

### Bucket Policy Example
```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::my-bucket/*"
    }
  ]
}
```
> ⚠️ This makes ALL objects in the bucket **publicly readable**. Only use for static websites.

### Block Public Access (4 Settings)
```
BlockPublicAcls         → Blocks new public ACLs
IgnorePublicAcls        → Ignores all existing public ACLs
BlockPublicPolicy       → Blocks new public bucket policies
RestrictPublicBuckets   → Restricts public access via any policy
```
> ✅ Keep ALL FOUR enabled unless you know what you're doing.

### Pre-Signed URLs
- Temporary URL granting access to a private object
- Expires after a set time (seconds to 7 days)
- Use case: let users download/upload without making bucket public

```bash
# Generate pre-signed URL (expires in 3600 seconds = 1 hour)
aws s3 presign s3://my-bucket/private-file.pdf --expires-in 3600
```

### Encryption Options
| Type | Who Manages Keys | Notes |
|------|-----------------|-------|
| **SSE-S3** | AWS | Default. S3 manages keys automatically |
| **SSE-KMS** | AWS KMS | You control the key, audit trail via CloudTrail |
| **SSE-C** | You | You provide the key with every request |
| **Client-side** | You | Encrypt before uploading |

## 🔁 Versioning

- Keeps **all versions** of every object
- Protects against accidental deletes and overwrites
- Once enabled → can only be **suspended**, not disabled
- Each version has a unique **Version ID**

### How Delete Works With Versioning
```
Without versioning:  DELETE file.txt  →  file.txt is GONE forever

With versioning:     DELETE file.txt  →  "delete marker" placed
                                         File appears deleted
                                         Old versions still exist
                                         Delete the marker → file restored!
```

> 💰 Cost warning: all versions are stored and billed. Use lifecycle rules to auto-delete old versions.

---

## ♻️ Lifecycle Policies

Automatically transition or expire objects based on age.

### Example: Move logs to cheaper storage
```
Day 0    → Upload to S3 Standard
Day 30   → Transition to Standard-IA        (save ~40%)
Day 90   → Transition to Glacier Flexible   (save ~80%)
Day 365  → Delete permanently
```

### Lifecycle Actions
```
Transition actions  →  Move object to cheaper storage class
Expiration actions  →  Permanently delete object after X days
```

## 🔔 S3 Event Notifications

Trigger actions when something happens in your bucket.

```
Event happens in S3  →  Sends notification to:
                          ├── SNS Topic  (email, SMS, fan-out)
                          ├── SQS Queue  (processing queue)
                          └── Lambda     (run code instantly)
```

### Supported Events
- `s3:ObjectCreated:*` — any upload/copy/multipart
- `s3:ObjectRemoved:*` — delete or delete marker
- `s3:ObjectRestore:*` — Glacier restore
- `s3:Replication:*` — replication events

## 🌍 Replication

### Cross-Region Replication (CRR)
- Copies objects to a bucket in **another AWS region**
- Use cases: compliance, lower latency for global users, disaster recovery
- Requires **versioning enabled** on both buckets

### Same-Region Replication (SRR)
- Copies objects within the **same region**
- Use cases: log aggregation, live replication between prod and test

```
Source bucket  →  replication rule  →  Destination bucket
(versioning ON)                        (versioning ON)
```

> ⚠️ Replication is NOT retroactive — only new objects after enabling are replicated.

## ⚡ Performance

### Multipart Upload
- Split large file into parts → upload in parallel → S3 reassembles
- **Required** for objects > 5 GB
- **Recommended** for objects > 100 MB
- If one part fails → retry only that part, not the whole file

### Transfer Acceleration
- Routes uploads through AWS **CloudFront edge locations**
- Useful for uploading from far away (e.g., India → us-east-1)
- Extra cost per GB transferred

### S3 Request Rate
```
3,500 PUT/COPY/POST/DELETE  per second per prefix
5,500 GET/HEAD              per second per prefix
```
> Use multiple prefixes to scale beyond these limits

## 🔍 S3 Select

- Query **inside** an object using SQL — without downloading the whole file
- Works on: CSV, JSON, Parquet (compressed or uncompressed)
- Up to **400% faster** and **80% cheaper** than full object retrieval

```sql
SELECT s.name, s.age FROM S3Object s WHERE s.age > 25
```

---

## 🔐 Object Lock

- Store objects using **WORM** model (Write Once, Read Many)
- Cannot be deleted or overwritten for a fixed period
- Two modes:
  - **Governance mode** — special IAM permissions can override
  - **Compliance mode** — NO ONE can delete, including root user

## 💸 Pricing Components

```
1. Storage          →  Per GB stored per month
2. Requests         →  Per 1,000 GET/PUT/LIST/DELETE requests
3. Data transfer    →  Data OUT to internet (IN is free)
4. Replication      →  Per GB replicated
5. Lifecycle        →  Transition requests cost money
6. Retrieval fees   →  Glacier classes charge for retrieval
```

> ✅ Free: Data transfer IN to S3, data transfer to EC2 in same region, S3 → CloudFront

## 🛠️ Useful AWS CLI Commands

```bash
# List all buckets
aws s3 ls

# List objects in a bucket
aws s3 ls s3://my-bucket/

# Upload a file
aws s3 cp file.txt s3://my-bucket/

# Upload entire folder
aws s3 sync ./local-folder s3://my-bucket/folder/

# Download a file
aws s3 cp s3://my-bucket/file.txt ./local/

# Delete a file
aws s3 rm s3://my-bucket/file.txt

# Delete all objects in bucket
aws s3 rm s3://my-bucket/ --recursive

# Generate pre-signed URL
aws s3 presign s3://my-bucket/file.txt --expires-in 3600

# Enable versioning
aws s3api put-bucket-versioning \
  --bucket my-bucket \
  --versioning-configuration Status=Enabled
```

## ❌ Common Mistakes

| Mistake | Fix |
|---------|-----|
| Disabling Block Public Access | Keep it ON unless hosting public static site |
| Not enabling versioning | Enable on important buckets from day one |
| Ignoring lifecycle rules | Set them up early — old versions cost money |
| Using root account | Always use IAM users with least privilege |
| No billing alarm | Set a $1 alert immediately after creating account |
| Storing secrets in S3 | Use AWS Secrets Manager or SSM Parameter Store |
| Same bucket for logs & app data | Separate buckets for separate concerns |

## 📝 Interview Questions — Quick Answers

**Q: What is the difference between S3 and EBS?**
> S3 = object storage, accessed via HTTP, infinite scale, not mountable. EBS = block storage, mounted to EC2, like a hard drive.

**Q: What is a pre-signed URL?**
> A temporary URL that grants time-limited access to a private S3 object without making it public.

**Q: What happens when you delete an object with versioning enabled?**
> S3 places a delete marker. The object appears deleted but all previous versions are preserved. Remove the delete marker to restore.

**Q: What is S3 Intelligent-Tiering?**
> S3 automatically moves objects between access tiers based on usage patterns, optimizing cost without performance impact.

**Q: Can you mount S3 like a file system?**
> Not natively. But tools like `s3fs` or AWS Storage Gateway allow mounting S3 as a file system.

**Q: What is the maximum size of an S3 object?**
> 5 TB. Files above 5 GB must use multipart upload.

**Q: What is CRR?**
> Cross-Region Replication — automatically copies objects from one bucket to a bucket in another AWS region.
