# 🌐 Static Website Hosting on AWS S3


---

## 📌 What is Static Website Hosting?

- S3 can serve **HTML, CSS, JS** files as a live website
- **No server needed** — S3 acts as the web server
- Perfect for: portfolios, landing pages, React/Vue builds, docs

| Type | Needs Server? | S3 Can Host? |
|------|---------------|--------------|
| Static (HTML/CSS/JS) | ❌ No | ✅ Yes |
| Dynamic (PHP/Node/Django) | ✅ Yes | ❌ No |

---

## 🪣 Step 1 — Create a Bucket

```
Bucket name : pythondevops-website-2026
Region      : ap-south-1
```

> ⚠️ Bucket name does NOT need to match your domain name

---

## 🔓 Step 2 — Disable Block Public Access

```
S3 → Bucket → Permissions → Block Public Access
→ Uncheck ALL 4 settings → Confirm
```

> ✅ Required so the world can read your website files

---

## ⚙️ Step 3 — Enable Static Website Hosting

```
S3 → Bucket → Properties → Static Website Hosting
→ Enable
→ Index document : index.html
→ Error document : error.html
→ Save
```

Your website URL will look like:
```
http://pythondevops-website-2026.s3-website.ap-south-1.amazonaws.com
```

---

## 📄 Step 4 — Create Website Files

**index.html**
```html
<!DOCTYPE html>
<html>
  <head>
    <title>My S3 Website</title>
  </head>
  <body>
    <h1>☁️ Hosted on AWS S3!</h1>
    <p>No servers. No EC2. Just S3. 🚀</p>
  </body>
</html>
```

**error.html**
```html
<!DOCTYPE html>
<html>
  <body>
    <h1>404 - Page Not Found</h1>
    <p>Oops! This page doesn't exist.</p>
  </body>
</html>
```

---

## 📤 Step 5 — Upload Files

**Console:**
```
S3 → Bucket → Upload → Add files → Select files → Upload
```

**CLI:**
```bash
# Upload single file
aws s3 cp index.html s3://pythondevops-website-2026/

# Upload entire folder
aws s3 sync ./my-website s3://pythondevops-website-2026/
```

---

## 🔐 Step 6 — Add Bucket Policy

```
S3 → Bucket → Permissions → Bucket Policy
```

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::pythondevops-website-2026/*"
    }
  ]
}
```

> ✅ Allows everyone to **read** your website files (GET only)

---

## 🌍 Step 7 — Open Your Website

```
S3 → Properties → Static Website Hosting → Bucket website endpoint
```

Paste URL in browser → **Your site is LIVE! 🎉**

---

## ⚡ Bonus — Deploy React/Vue Build

```bash
# Build your React app
npm run build

# Sync to S3 (--delete removes old files)
aws s3 sync ./build s3://pythondevops-website-2026/ --delete
```

## ❌ Common Issues & Fixes

| Problem | Cause | Fix |
|---------|-------|-----|
| 403 Forbidden | Block Public Access ON | Disable all 4 settings |
| 403 Forbidden | No bucket policy | Add `s3:GetObject` policy |
| 404 Not Found | Wrong index document | Set `index.html` in static hosting |
| URL shows XML | Static hosting not enabled | Enable in Properties |
| Old files showing | Browser cache | Hard refresh `Ctrl+Shift+R` |

## 🔑 Key Points

```
✅ S3 hosts static websites — HTML, CSS, JS only
✅ Must disable Block Public Access
✅ Must add bucket policy → s3:GetObject
✅ Set index.html as index document
✅ Two different URLs:

Object URL  → https://bucket.s3.region.amazonaws.com/index.html
Website URL → http://bucket.s3-website.region.amazonaws.com
```

## 📝 Interview Questions

**Q: Can S3 host dynamic websites?**
> No. S3 is for static files only. Use EC2 or Lambda for dynamic sites.

**Q: Difference between S3 URL and S3 Website URL?**
> S3 URL = direct object access. Website URL = serves with index/error document support.

**Q: How to make the website faster?**
> Put **CloudFront** in front of S3 — global CDN + HTTPS. → Video 05! 😉
