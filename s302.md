# ☁️ AWS S3 — Core Concepts (Quick Theory)


---

## 🪣 Bucket

- A bucket is a container that stores your data in S3
- Think of it like a hard drive in the cloud
- Every file must be stored inside a bucket

Key Points:
- Bucket name must be globally unique
- Bucket is created in a specific region (e.g. ap-south-1)

Example:
Bucket Name: pythondevops-s3-demo

---

## 📦 Object

- Any file stored in S3 is called an object
- It can be image, video, PDF, JSON, backup, etc.

Key Points:
- Max size of a single object = 5 TB
- Each object contains:
  - File data
  - Metadata
  - Key (name of file)

Example:
photo.jpg
backup.zip
data.json

---

## 🔑 Key

- A key is the unique name of an object inside a bucket

Important:
- S3 does NOT have real folders
- "/" is just part of the name (not a directory)

Example:
profile/user1.jpg
logs/2026/march/app.log

---

## 🌐 URL Structure

https://bucket-name.s3.region.amazonaws.com/key

Example:
https://pythondevops-s3-demo.s3.ap-south-1.amazonaws.com/profile/user1.jpg

Breakdown:
- bucket-name → your bucket
- region → where bucket is hosted
- key → file name/path

---

## ⚠️ Important Rule

Bucket name must be globally unique across ALL AWS

Wrong:
test-bucket
mybucket

Correct:
pythondevops-s3-demo-2026

Tip:
Always add your name, project, or year to make it unique
